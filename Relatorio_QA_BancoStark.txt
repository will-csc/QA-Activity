PARTE 05 — Bug Report (atividade)

Template — Bug Report 1
Título: Sistema permite depósito com valor zero
Severidade (Baixa/Média/Alta): Baixa
Prioridade (Baixa/Média/Alta): Baixa
Ambiente: Windows / Python 3.x / Banco do Stark
Descrição:
O sistema aceita depósitos de valor 0, gerando uma entrada no extrato sem alteração de saldo, o que é um comportamento desnecessário e viola a regra de negócio "Depósito: valor deve ser > 0".
Passos para reproduzir:
1. Instanciar o BancoDoStark e criar uma conta (ex: 'tony')
2. Executar banco.depositar('tony', 0)
3. Verificar a saída e o extrato
Resultado esperado: O sistema deveria rejeitar o depósito com uma mensagem de erro ou aviso, pois o valor deve ser maior que 0.
Resultado obtido: O sistema processou o depósito com sucesso ("OK - Depósito realizado: 0").
Evidência:
Teste N1 - Depósito com valor zero
OK - Depósito realizado: 0
Impacto/Risco: Baixo. Poluição do extrato com transações nulas.

Template — Bug Report 2
Título: Sistema permite depósito com valor negativo (reduz saldo)
Severidade (Baixa/Média/Alta): Alta
Prioridade (Baixa/Média/Alta): Alta
Ambiente: Windows / Python 3.x / Banco do Stark
Descrição:
O sistema aceita depósitos com valores negativos, o que causa a redução do saldo do cliente indevidamente, funcionando como um saque não autorizado. Viola a regra "Depósito: valor deve ser > 0".
Passos para reproduzir:
1. Instanciar o BancoDoStark e criar uma conta (ex: 'tony' com saldo inicial)
2. Executar banco.depositar('tony', -100)
3. Verificar o saldo da conta
Resultado esperado: O sistema deve rejeitar valores negativos no depósito.
Resultado obtido: O sistema processou o depósito negativo ("OK - Depósito realizado: -100") e o saldo foi reduzido.
Evidência:
Teste N2 - Depósito negativo
OK - Depósito realizado: -100
Impacto/Risco: Alto. Perda financeira para o cliente ou inconsistência contábil grave.

Template — Bug Report 3
Título: Sistema permite transferência com valor negativo (fraude financeira)
Severidade (Baixa/Média/Alta): Crítica
Prioridade (Baixa/Média/Alta): Alta
Ambiente: Windows / Python 3.x / Banco do Stark
Descrição:
O sistema permite transferências com valores negativos. Isso inverte a lógica da transferência: a conta de origem RECEBE o valor (subtração de negativo vira soma) e a conta de destino PERDE o valor (soma de negativo vira subtração). Isso permite criar dinheiro na conta de origem sem lastro.
Passos para reproduzir:
1. Ter duas contas existentes ('tony' e 'pepper')
2. Executar banco.transferir('tony', 'pepper', -50)
3. Verificar o saldo de 'tony' (origem)
Resultado esperado: O sistema deve impedir transferências com valor <= 0.
Resultado obtido: A transferência foi realizada ("OK - Transferência realizada: -50"). O saldo de 'tony' aumentou em 50 e o de 'pepper' diminuiu em 50.
Evidência:
Teste Extra 2 - Transferência com valor negativo
OK - Transferência realizada: -50
Impacto/Risco: Crítico. Permite fraude onde um usuário pode "roubar" dinheiro de outro ou inflar seu próprio saldo apenas iniciando uma transferência negativa.
